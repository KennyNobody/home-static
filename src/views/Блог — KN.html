Vue.js шпаргалка

Оригинал: Vue.js Cheat Sheet

Директивы

Директивы и атрибуты обозначаются префиксом v- :

v-text    // вставляет значение как текстовое содержимое элемента
v-html    // вставляет значение как HTML разметку
v-if      // показывать элемент, только если условие истинно
v-else    // показывать элемент, если предыдущее условие ложно
v-else-if // промежуточная конструкция между if и else
v-show    // аналогично if, но лишь добавляет display: none, если условие == false, оставляя элемент в DOM
v-for     // отрисовывает каждый элемент итерируемого массива или объекта
v-on      // отслеживает DOM-события
v-bind    // реактивно связывает атрибуты тега или входной параметр компонента с выражением
v-model   // двусторонне связывает input с переменной
v-once    // однократно отрисовывает элемент или компонент

Директивы v-bind и v-on имеют укороченный формат записи:

<a v-bind:href="url">...</a>
<a :href="url">...</a>

<a v-on:click="doSomething">...</a>
<a @click="doSomething">...</a>

Примеры v-if/v-else/v-else-if:

<div v-if="type === A">
    это A
</div>
<div v-else-if="type === B">
    это B
</div>
<div v-else-if="type === C">
    это C
</div>
<div v-else>
    это что-то другое
</div>

Модификаторы событий

Для примера используется событие click, но они аналогичные и для любого другого:

v-on:click.native  // подписаться на нативное событие на корневом элементе компонента
v-on:click.stop    //  предотвращает всплытие события
v-on:click.passive //  вызов обработчика события DOM с опцией { passive: true }
v-on:click.capture //  добавит подписку в режиме capture
v-on:click.self    //  вызовет обработчик, только если событие возникло непосредственно на этом элементе
v-on:click.once    //  вызовет обработчик не больше одного раза
v-on:click.prevent //  вызовет event.preventDefault()

Методы объекта Vue


Vue.extend    // Создаёт “подкласс” базового конструктора Vue. Принимает параметром объект с опциями нового компонента
Vue.nextTick  // Вызов функции callback при следующем цикле обновления DOM. Используйте для работы с обновлённым DOM после изменения данных
Vue.set       // Добавляет реактивное свойство к объекту
Vue.delete    // Удаляет свойство из объекта
Vue.directive // Регистрирует новую глобальную директиву или возвращает уже зарегистрированную
Vue.filter    // Регистрирует новый глобальный фильтр или возвращает уже зарегистрированный 
Vue.component // Регистрирует глобальный компонент или возвращает уже зарегистрированный

Vue.compile   // Компилирует строковый шаблон в render-функцию
Vue.version   // Возвращает версию Vue в виде строки

Опции объекта Vue

При инициализации Vue мы создаем объект вида:

let app = new Vue ({
});

Далее этот объект может принимать различные значения

data      // объект с данными экземпляра Vue, которые будут реактивными. Все значения должны быть добавлены до момента инициализации 
props     // список входных параметров, по которым разрешено получение данных из родительского компонента
propsData // значения по умолчанию для props, используются только для тестирования
method    // устанавливает методы, которые будут добавляться к экземпляру Vue
computed  // аналогично, но с внутренним кешированием, функцию вызываем как шаблон (без скобок)
watch     // позволяет отслеживать свойства и вызывать функции при их изменении

Пример:

var vm = new Vue ({
    el: "#example",
    data: {
        message: "Привет!"
    },
    methods: {
        reverseMessageAsMethod: function(){
            return this.message.split("").reverse("").join("");
        }
    },
    computed {
        // вычисленный геттер
        reversedMessage() {
            return this.message.split("").reverse("").join("");
        }
    }
});

console.log(vm.reverseMessageAsMethod) // => 'olleH';
vm.message = "Goodbye;"
console.log(vm.reversedMessage) // => 'eybdooG';
